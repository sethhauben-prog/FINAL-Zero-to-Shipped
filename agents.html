💰 Understanding API Costs — Before You Build Anything ▾

This is the most important thing to understand before building agents. Unlike web apps that run on free Vercel hosting, agents call APIs every time they run — and those calls cost money. Understanding costs upfront prevents nasty surprises.

1
How Claude API pricing works
You pay per token — roughly per word. Claude Sonnet 4 costs approximately $0.003 per 1,000 input tokens and $0.015 per 1,000 output tokens. A typical agent call costs about $0.01–$0.05. Check current pricing at anthropic.com/pricing.
2
Estimate your agent's monthly cost before deploying
The formula: (runs per day) × (cost per run) × 30. An agent that runs 24 times per day (hourly) at $0.02 per run = $14.40/month. Always calculate this before setting a schedule.
3
Set a spending limit before you start
In the Anthropic Console, go to Billing → Usage Limits and set a monthly hard limit — e.g. $20. This caps your exposure if an agent runs in an unexpected loop.
4
Keep prompts short and efficient
Every word in your system prompt costs money, multiplied by every run. Ask Claude: "Rewrite this system prompt to be as concise as possible while keeping all the important instructions."
✍️ Prompt Engineering for Agents — Different from Chatting ▾

Writing prompts for an autonomous agent is very different from chatting with Claude. In a chat, you can clarify and correct. In an agent, the prompt runs unattended — every ambiguity becomes a bug.

1
Give the agent a clear role and scope
Start every agent system prompt with: what the agent is, what it should do, and — crucially — what it should NOT do.
You are an email response assistant for [Business Name]. Your job: read incoming customer support emails and draft a helpful reply. ONLY respond to questions about our products and services. DO NOT make promises about refunds, timelines, or pricing without checking. If you're unsure how to respond, say "I'll need to check on this" instead of guessing.
2
Specify the output format exactly
Agents pass their output to the next step — a webhook, an email, a database. If the format varies, downstream steps break. Tell the agent exactly what to return: "Respond with only a JSON object with keys: reply_text (string), confidence (1-10), needs_human_review (boolean). No other text."
3
Test edge cases before going live
Before turning on an automated agent, manually test at least 5 edge cases: empty input, very long input, input in another language, rude or nonsensical input. Ask Claude: "What are 5 edge cases that might cause this agent to behave unexpectedly?"
🎯 What You're Building — and What an Agent Actually Is ▾

An agent is an AI that doesn't just answer questions — it takes action. Instead of you typing a prompt and reading a response, an agent watches for a trigger, processes it using AI, and does something with the result — all without you touching a keyboard.

Add another Gmail action. Choose "Send Email". Set To to the sender's email. Set Body to Claude's output from the previous step.

Click Test in Zapier. Review the output — does Claude's reply make sense? When satisfied, toggle the Zap to On. Your agent is now live.

🛡️ Error Handling — Agents Fail Silently Without This ▾

An agent running unattended can fail for hours without anyone noticing. Without error handling, you'll never know. With it, failures surface immediately.

1
Wrap every external call in try/except
Ask Claude Code: "Add try/except error handling around every external API call. If a call fails, log the error with a timestamp and continue rather than crashing the whole agent."
2
Validate Claude's output before using it
When your agent expects Claude to return JSON, always validate it before passing it downstream. Ask Claude Code: "Before using Claude's response, check that it matches the expected format. If it doesn't, retry once. If it fails again, log the bad output and skip this run."
3
Add a dead agent alert
Ask Claude Code: "If the agent hasn't successfully completed a run in 3 hours, send me an alert email." This catches situations where the agent crashes silently.
4
Handle empty or poor search results gracefully
Never send Claude an empty results set — it will hallucinate content. Always check: "If the search returns fewer than 2 results, skip this run and log 'insufficient results' rather than generating a digest."

Multi-Agent Pipeline

A multi-agent pipeline where specialized Claude agents pass work to each other. One agent researches, another writes, a third reviews and scores — if the score is too low, the pipeline retries automatically.

You'll orchestrate the pipeline in Python, passing structured JSON state between agents and implementing quality gates with retry logic so the system self-corrects without human intervention.

Create three specialized agents and wire them together in a pipeline.

Create pipeline.py with the three agents:

import anthropic
import json

client = anthropic.Anthropic()

def research_agent(topic: str) -> dict:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        messages=[{"role": "user", "content": f"""Research this topic and return JSON with:
- key_points: list of 5 facts
- context: 2-3 sentence background
- sources_needed: list of source types

Topic: {topic}
Return only valid JSON."""}]
    )
    return json.loads(response.content[0].text)

def writer_agent(research: dict, topic: str) -> str:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=2048,
        messages=[{"role": "user", "content": f"""Write a 300-word article on '{topic}' using this research:
{json.dumps(research, indent=2)}
Include a headline. Write for a general audience."""}]
    )
    return response.content[0].text

def reviewer_agent(article: str, topic: str) -> dict:
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=512,
        messages=[{"role": "user", "content": f"""Review this article on '{topic}' and return JSON with:
- score: integer 1-10
- issues: list of specific problems
- verdict: "pass" if score >= 7, else "fail"

Article: {article}
Return only valid JSON."""}]
    )
    return json.loads(response.content[0].text)

def run_pipeline(topic: str):
    print(f"[Pipeline] Topic: {topic}")
    research = research_agent(topic)
    article = writer_agent(research, topic)
    review = reviewer_agent(article, topic)
    print(f"Score: {review['score']}/10 — {review['verdict']}")
    return article, review

if __name__ == "__main__":
    article, review = run_pipeline("the rise of AI agents")
    print(article)

Wrap the pipeline in a retry loop that re-runs the writer if the reviewer scores below 7.

def run_pipeline_with_retry(topic: str, max_attempts: int = 3):
    print(f"[Pipeline] Max {max_attempts} attempts")
    research = research_agent(topic)  # Research once, reuse

    for attempt in range(1, max_attempts + 1):
        print(f"\n[Attempt {attempt}/{max_attempts}]")
        article = writer_agent(research, topic)
        review = reviewer_agent(article, topic)
        print(f"  Score: {review['score']}/10 — {review['verdict']}")

        if review['verdict'] == 'pass':
            print(f"✓ Passed on attempt {attempt}")
            return article, review, attempt

        if attempt < max_attempts:
            print(f"  Issues: {review['issues']} — retrying...")

    print("⚠ Max attempts reached.")
    return article, review, max_attempts

if __name__ == "__main__":
    article, review, attempts = run_pipeline_with_retry("the rise of AI agents")
    print(f"\nFinal article (after {attempts} attempt(s)):\n{article}")

Model Context Protocol (MCP) is an open standard that lets AI models connect to external tools and data sources through a common interface. An MCP server exposes a set of tools the model can call — like reading a file, querying a database, or fetching a webpage.

Instead of writing one-off integration code for every tool, you configure MCP servers once and Claude can use them automatically. In this project you'll connect Claude Code to Supabase so your agent can store and retrieve memories without you writing the query code yourself.

An agent that actually remembers things across conversations. When you tell it something important, it stores a vector embedding in Supabase. When you ask a question later, it searches those memories semantically and includes the most relevant ones in its context.

You'll also wire up the Supabase MCP server so Claude Code can query your database directly using natural language — no raw SQL needed.

Create a Supabase project and enable pgvector for semantic memory search.

-- Enable vector extension
create extension if not exists vector;

-- Memory table
create table memories (
  id uuid primary key default gen_random_uuid(),
  content text not null,
  embedding vector(1536),
  created_at timestamptz default now()
);

-- Semantic search function
create or replace function match_memories(
  query_embedding vector(1536),
  match_threshold float default 0.7,
  match_count int default 5
)
returns table (id uuid, content text, similarity float)
language sql stable as $$
  select id, content,
    1 - (embedding <=> query_embedding) as similarity
  from memories
  where 1 - (embedding <=> query_embedding) > match_threshold
  order by similarity desc
  limit match_count;
$$;
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=your-service-role-key
ANTHROPIC_API_KEY=your-key-here

Write the agent that stores and retrieves memories using vector embeddings.

import os
from anthropic import Anthropic
from supabase import create_client
from dotenv import load_dotenv

load_dotenv()
client = Anthropic()
supabase = create_client(os.environ["SUPABASE_URL"], os.environ["SUPABASE_SERVICE_KEY"])

def embed(text: str) -> list[float]:
    """Generate a vector embedding for text."""
    # Use Voyage embeddings via the Anthropic Batches API or a local model.
    # For quick prototyping, ask Claude Code: "add real embeddings using voyage-3"
    import hashlib, random
    seed = int(hashlib.md5(text.encode()).hexdigest(), 16) % (2**32)
    rng = random.Random(seed)
    return [rng.gauss(0, 1) for _ in range(1536)]

def store_memory(content: str):
    supabase.table("memories").insert({
        "content": content,
        "embedding": embed(content)
    }).execute()
    print(f"[Memory stored] {content[:60]}...")

def search_memories(query: str, limit: int = 3) -> list[str]:
    result = supabase.rpc("match_memories", {
        "query_embedding": embed(query),
        "match_count": limit
    }).execute()
    return [row["content"] for row in (result.data or [])]

def chat(user_message: str, history: list) -> str:
    memories = search_memories(user_message)
    memory_context = ""
    if memories:
        memory_context = "\n\nRelevant memories:\n" + "\n".join(f"- {m}" for m in memories)

    history.append({"role": "user", "content": user_message})
    response = client.messages.create(
        model="claude-sonnet-4-6",
        max_tokens=1024,
        system=f"You are a helpful assistant with persistent memory.{memory_context}",
        messages=history
    )
    reply = response.content[0].text
    history.append({"role": "assistant", "content": reply})
    return reply

if __name__ == "__main__":
    history = []
    print("Memory Agent ready. Type 'remember: ...' to store a memory. Type 'quit' to exit.\n")
    while True:
        user_input = input("You: ").strip()
        if user_input.lower() == "quit":
            break
        if user_input.lower().startswith("remember:"):
            store_memory(user_input[9:].strip())
            print("Agent: Got it, I'll remember that.\n")
        else:
            print(f"Agent: {chat(user_input, history)}\n")

An autonomous agent running unsupervised needs different evaluation than an interactive one. Before deploying, ask yourself:

  • Worst case? If the agent makes a mistake, what's the maximum damage? Can it be reversed?
  • Tight feedback loop? Will you know quickly if something goes wrong?
  • Correctly scoped tools? Each tool should do exactly one thing. Avoid broad write permissions.
  • Edge cases tested? Empty inputs, API failures, rate limits, unexpected data formats.

Start with read-only tools and a human approval step. Add write tools and full autonomy only after the agent proves reliable.

Every autonomous agent needs explicit trust boundaries — a clear definition of what it can do without human approval.

Low trust (require approval): sending emails, posting to social media, making purchases, deleting data, calling paid APIs repeatedly.

High trust (run autonomously): reading files, searching the web, summarizing content, writing to a staging environment, logging to a database.

In this project the agent drafts and prints the email before sending, giving you a chance to review. In production you'd add an approval webhook or confirmation step.

A fully autonomous business agent that runs on a schedule in the cloud. Every morning it scrapes your industry's news, summarizes the top stories, identifies business opportunities, drafts an email briefing, and sends it — all without you touching it.

You'll build it locally with tool use, then deploy to Railway as a cron job that fires every day at 7 AM.

Build the agent with tool use so it can search the web, scrape content, and send email.

import os, json, requests
from anthropic import Anthropic
from dotenv import load_dotenv

load_dotenv()
client = Anthropic()

# --- Tools ---

def search_news(query: str, num_results: int = 5) -> list:
    r = requests.post("https://google.serper.dev/news",
        headers={"X-API-KEY": os.environ["SERPER_API_KEY"]},
        json={"q": query, "num": num_results})
    return r.json().get("news", [])

def scrape_page(url: str) -> str:
    r = requests.get(f"https://r.jina.ai/{url}", timeout=15)
    return r.text[:4000]

def send_email(to: str, subject: str, body: str) -> str:
    r = requests.post("https://api.resend.com/emails",
        headers={"Authorization": f"Bearer {os.environ['RESEND_API_KEY']}",
                 "Content-Type": "application/json"},
        json={"from": os.environ["EMAIL_FROM"], "to": [to],
              "subject": subject, "text": body})
    return f"Email sent: {r.status_code}"

TOOLS = [
    {"name": "search_news", "description": "Search recent news on a topic",
     "input_schema": {"type": "object",
       "properties": {"query": {"type": "string"}, "num_results": {"type": "integer"}},
       "required": ["query"]}},
    {"name": "scrape_page", "description": "Scrape a webpage's text content",
     "input_schema": {"type": "object",
       "properties": {"url": {"type": "string"}}, "required": ["url"]}},
    {"name": "send_email", "description": "Send the daily briefing email",
     "input_schema": {"type": "object",
       "properties": {"to": {"type": "string"}, "subject": {"type": "string"},
                      "body": {"type": "string"}},
       "required": ["to", "subject", "body"]}}
]
TOOL_MAP = {"search_news": search_news, "scrape_page": scrape_page, "send_email": send_email}

# --- Agent loop ---

def run_agent(topic: str):
    print(f"[Agent] Starting briefing for: {topic}")
    messages = [{"role": "user", "content": f"""You are an autonomous business intelligence agent.

Your task:
1. Search for the latest news on: {topic}
2. Scrape 2-3 of the most relevant articles
3. Identify top 3 business opportunities or risks
4. Draft and send a briefing email to {os.environ['EMAIL_TO']}
   Subject: "Daily Briefing: {topic}"
   Include: top stories, key insights, opportunities/risks

Complete this task autonomously using your tools."""}]

    while True:
        response = client.messages.create(
            model="claude-sonnet-4-6", max_tokens=4096,
            tools=TOOLS, messages=messages)
        messages.append({"role": "assistant", "content": response.content})

        if response.stop_reason == "end_turn":
            print("[Agent] Done.")
            break

        tool_results = []
        for block in response.content:
            if block.type == "tool_use":
                print(f"  [Tool] {block.name}")
                result = TOOL_MAP[block.name](**block.input)
                tool_results.append({"type": "tool_result", "tool_use_id": block.id,
                    "content": json.dumps(result) if not isinstance(result, str) else result})
        messages.append({"role": "user", "content": tool_results})

if __name__ == "__main__":
    run_agent("AI and SaaS startups")
